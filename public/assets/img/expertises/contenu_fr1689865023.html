Ajouter du code hébergé localement à GitHub

Découvrez comment ajouter du code source ou des référentiels existants à GitHub à partir de la ligne de commande à l'aide de la CLI GitHub ou des commandes Git. Ensuite, partagez votre code et invitez d'autres personnes à travailler avec vous.

À propos de l'ajout de code source existant à GitHub

Si vous avez du code source ou des référentiels existants stockés localement sur votre ordinateur ou votre réseau privé, vous pouvez les ajouter à GitHub en tapant des commandes dans un terminal. Vous pouvez le faire en tapant directement les commandes Git ou en utilisant l'interface de ligne de commande GitHub.

GitHub CLI est un outil open source permettant d'utiliser GitHub à partir de la ligne de commande de votre ordinateur. GitHub CLI peut simplifier le processus d'ajout d'un projet existant à GitHub à l'aide de la ligne de commande. Pour en savoir plus sur GitHub CLI, consultez « À propos de GitHub CLI ».

Astuce : si vous êtes plus à l'aise avec une interface utilisateur pointer-cliquer, essayez d'ajouter votre projet avec GitHub Desktop. Pour plus d'informations, consultez « Ajouter un référentiel à partir de votre ordinateur local à GitHub Desktop » dans l' aide de GitHub Desktop .

Avertissement : Jamais git add, commit, ou pushdes informations sensibles vers un référentiel distant. Les informations sensibles peuvent inclure, mais sans s'y limiter :

  • Mots de passe
  • Clés SSH
  • Clés d'accès AWS
  • Clés API
  • Numéros de carte de crédit
  • NIP

Pour plus d'informations, consultez " Suppression de données sensibles d'un référentiel ".

Ajouter un référentiel local à GitHub avec GitHub CLI

  1. Dans la ligne de commande, accédez au répertoire racine de votre projet.

  2. Initialisez le répertoire local en tant que référentiel Git.

    git init -b main

  3. Organisez et validez tous les fichiers de votre projet.

    git add . && git commit -m "initial commit"

  4. Pour créer un dépôt pour votre projet sur GitHub, utilisez la gh repo createsous-commande. Lorsque vous y êtes invité, sélectionnez Transférer un référentiel local existant vers GitHub et entrez le nom souhaité pour votre référentiel. Si vous souhaitez que votre projet appartienne à une organisation au lieu de votre compte utilisateur, spécifiez le nom de l'organisation et le nom du projet avec organization-name/project-name.

  5. Suivez les invites interactives. Pour ajouter la télécommande et pousser le référentiel, confirmez oui lorsqu'on vous demande d'ajouter la télécommande et de pousser les commits vers la branche actuelle.

  6. Sinon, pour ignorer toutes les invites, indiquez le chemin d'accès au référentiel avec l' --sourceindicateur et transmettez un indicateur de visibilité ( --public, --privateou --internal). Par exemple, gh repo create --source=. --public. Spécifiez une télécommande avec le --remotedrapeau. Pour pousser vos commits, passez le --pushdrapeau. Pour plus d'informations sur les arguments possibles, consultez le manuel GitHub CLI .

Ajouter un référentiel local à GitHub à l'aide de Git

  1. Create a new repository on GitHub.com. To avoid errors, do not initialize the new repository with README, license, or gitignore files. You can add these files after your project has been pushed to GitHub.

  2. Open TerminalTerminalGit Bash.

  3. Change the current working directory to your local project.

  4. Use the init command to initialize the local directory as a Git repository. By default, the initial branch is called main.

    If you’re using Git 2.28.0 or a later version, you can set the name of the default branch using -b.

    $ git init -b main

    If you’re using Git 2.27.1 or an earlier version, you can set the name of the default branch using && git symbolic-ref HEAD refs/heads/main.

    $ git init && git symbolic-ref HEAD refs/heads/main

  5. Add the files in your new local repository. This stages them for the first commit.

    $ git add .
# Adds the files in the local repository and stages them for commit. To unstage a file, use 'git reset HEAD YOUR-FILE'.

  6. Commit the files that you've staged in your local repository.

    $ git commit -m "First commit"
# Commits the tracked changes and prepares them to be pushed to a remote repository. To remove this commit and modify the file, use 'git reset --soft HEAD~1' and commit and add the file again.

  7. At the top of your repository on GitHub.com's Quick Setup page, click to copy the remote repository URL.

  8. In Terminal, add the URL for the remote repository where your local repository will be pushed.

    $ git remote add origin <REMOTE_URL>
# Sets the new remote
$ git remote -v
# Verifies the new remote URL

  9. Push the changes in your local repository to GitHub.com.

    $ git push -u origin main
# Pushes the changes in your local repository up to the remote repository you specified as the origin
